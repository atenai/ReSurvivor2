# ReSurvivor2

「Reサバイバー2&モーションビュアー」

（概要・テーマ）
この作品は、Re:サバイバーの続編として開発中の作品です。
伝令である主人公が情報を目的地まで戦場をくぐりぬけて行く「伝令」というテーマで制作しています。
またゲームシステムは前作の横スクロールシューティングゲームからTPSゲームへの進化を目標にしています。
その開発過程でアニメーションのチェック用にモーションビュアーを作成しました。

(開発時期・期間)
2023年11月～2025年2月(25ヶ月)

（開発ツール）
・ゲームエンジン：Unity(バージョン2022.3.62f2)
・言語：C# 
・開発環境：Visual Studio Code 
・3D データ： Unityのアセットストアを使用しています。
・2D データ : フリー素材を使用しています。

（チームメンバー）
開発チーム（1人）
・プログラマー1人（担当）

その中にある、「ReSurvivor2.exe」というアプリケーションをダブルクリックしてください。

(命名規則)
・変数名は「小文字」から始める(キャメルケース)
(例)
int test = 0;

・関数名は「大文字」から始める(パスカルケース)
(例)
void Test();

・引数名は「小文字」から始める(キャメルケース)
(例)
void Test(int test);

・プロパティ名は「取得する変数名を大文字」にしたものにする(パスカルケース)
(例)
int test = 0;
public int Test
{
	set{ test = value; } //値の代入
	get { return test; } //外部に値を返す
}

・bool型の変数または関数は頭文字に必ず「is」(変数)「Is」(関数)をつける
(例)
bool isHoge;
(例)
bool IsHoge();

・UnityActionなどイベントを使用するものは頭文字に「on」(変数)「On」(関数)をつける   
(例)
UnityAction onFinished;
(例)
UnityAction OnFinished();

・デバッグ用の変数または関数は頭文字に「debug」(変数)「Debug」(関数)と付け、長く細かく書いた命名にする
(例)
int debugTest = 0;
void DebugTest();
	
・インターフェース「interface」は「I」から始める
(例)
IDamage
	
・アブストラクト「abstract」は「〜Base」とBaseを付ける
(例)
GunBase
	
・「const」や「readonly」ような定数は「パスカルスネークケース」を使う
(例)
readonly int Test_Hoge = 1;
	
・構造体「struct」は大文字から始める（パスカルケース）
(例)
struct TextProperty
{
	
}	

・Unityのアニメーション用変数名は「型の最初の文字_小文字」にする(システムハンガリアン記法)
(例)
f_test
